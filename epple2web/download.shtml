Downloading and Installing

Epple ][ Program—The Hardware (Emulated)

The Epple ][ Program is the emulator itself, which emulates the Apple ][ or ][ plus hardware, as well as the peripheral card hardware. Download the file for your platform (binary, ready to run), or source if you want to compile it yourself (see Building). For binary downloads, install the package as appropriate: for Linux,
Redhat 

sudo rpm -i epple2-1.0-1.i386.rpm
or Debian
sudo dpkg -i epple2_1.0-2_i386.deb
 For Windows, allow the browser to run the epple2-1.0.msi file, or else save it and double-click it.

System ROMs—The Main System Firmware (Demo or Real)

Since the Epple ][ program just emulates hardware, you will also need to download some system software for it to run. This is commonly called "firmware" or "ROMs," because it is stored in the read-only memory area of the emulated system. The original system software on the Apple ][ machines is copyright by Apple, and is proprietary. If you own an actual Apple ][ or ][ plus machine, you can copy the firmware from it ($D000-$FFFF) (into a file on your PC) and use it. Alternatively, you can download the firmware images from the Apple ][ Library. In any case, you must respect the terms of the copyright (owned by Apple); this page is not intended to give you any legal advice.

If you do not want to download the proprietary firmware, you can still use the emulator (albeit in a restricted manner) with the free (GPLv3) "System ROM (Demo)" on this page, above. Download the file for your platform and install as appropriate. You can choose either binary format (read to run), or source if you want to assemble it yourself (using the cc65 Assembler). The Demo System ROM only provides commands to dump or set memory bytes, or run a program in memory. It is not compatible with any Apple software, nor can it read from disks. It's just a free simple demo system to make the emulator do something.

Card ROMs—The Peripheral Card Firmware

If you are going to use the Disk ][ Controller peripheral card (which you almost certainly will, if you want to read any floppy disk images), you will need to have to original ROMs from Apple. No demo of the Disk ROMs are provided here; you will need to use the original ROMs, which can be obtained from an original machine (by copying it from $C600-$C6FF into a file on the PC), or from the Apple ][ Library. In any case, you must respect the terms of the copyright (owned by Apple); this page is not intended to give you any legal advice. (And, of course, you will need the real Apple System ROM, as noted in the previous section, not the Demo System ROM)

If you are using the clock, the standard output, or the standard input peripheral cards, you will need to download their firmware as well, which is provided on this page, above, at "Card ROMs". You can choose either binary format (ready to run) or source format (to assemble with the cc65 Assembler). If you are not going to use the stdin, stdout, or clock cards, then you do not need to download this.

Configuring

After downloading and installing the Epple ][ Program, some System ROM (demo or real), and optionally the peripheral card ROMs you want to use, you will need to configure the program. Configuring is concerned primarily with telling the program what peripheral cards to use, and where the ROM files are located (and what memory addresses to load them at). There is a section that describes in detail how to configure the program.
